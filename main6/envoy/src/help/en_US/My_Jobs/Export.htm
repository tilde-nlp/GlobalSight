Export

Summary:

Exports files from GlobalSight.  The process of exporting does not remove any data from GlobalSight.  In the case of Primary Target Files, it simply retrieves the localized data from the database and re-constructs the original source file with the new localized data.  For Secondary Target Files, it simply copies the files into the export location.

Target Locales

Place a check mark next to the target locales to be exported.  If you are exporting from My Activites, or from My Jobs as a Workflow Manager, only one target locale is shown.

 

Under each target locale, each file is also listed.  Place a check mark next to the files to export.

Locale-Specific Sub Directory

You can change the name of the root directory to which the file exports.  However, the path of the file still reflects the path of the source file, ignoring the first directory.  If the source path were \myFiles\someDocuments\newFile.doc, the file would by default be exported to \fr_FR\someDocuments\newFile.doc.  If the locale-specific sub directory were changed to "exported-fr_FR", the file would export to \exported-fr_FR\someDocuments\newFile.doc.

Export Location

Select the location for export.  By default files are exported into the same \docs directory from which they were imported (CXE Docs Dir).

Localized

Indicates whether the final activity in the workflow has been completed.

 

Special Note: When a PDF file is exported, it is exported as a .doc (Word) file.  This is because the PDF format does not store words or sentences as a normal file does.  In order to import a PDF, OCR (optical character recognition) is used to first convert the file into a Word document.  After export, the .doc file can be used to generate a PDF.

Character Encoding

Determines the character coding for export of the localized file.  For Microsoft Office and PDF files, the encoding will always be UTF-8.

UTF-8 Byte Order Mark (BOM)

Select method on how to process UTF-8 Byte Order Mark (BOM) information for original file.
  1.Use File Profile Settings
  2.Preserve; do not add if not originally present
  3.Preserve; add if not originally present
  4.Remove if present

Populate Source Into Un-Translated Target (XLF/XLZ only)

If source file is XLF/XLZ, for un-translated segment with empty "target" originally, this option determines if populate source content into target in exported file. Default no.

Primary Target Files

Select the files to be exported.  Original file(s) are reintegrated with the localized data and the resulting localized file(s) are written to the selected Export Location.

Secondary Target Files

Select the files to be exported.  Secondary Target file(s) are written to the selected Export Location.

Buttons

 

Cancel

Cancels the export.

Export

Initiates the export.  You will receive an email when the export is complete.
