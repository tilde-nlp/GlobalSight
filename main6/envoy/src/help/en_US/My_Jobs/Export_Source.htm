Export Source (optional)

Summary:

Exports files from GlobalSight with GSA Added content, and marks content as GSA Deleted.  Jobs containing exported files are cancelled, however, all current translations are committed to the TM.

Locale-Specific Sub Directory

You can change the name of the root directory to which the file exports.  However, the path of the file still reflects the path of the source file, ignoring the first directory.  If the source path were \myFiles\someDocuments\newFile.doc, the file would by default be exported to \en_US\someDocuments\newFile.doc.  If the locale-specific sub directory were changed to "exported-en_XX", the file would export to \exported-en_XX\someDocuments\newFile.doc.

Export Location

Select the location for export.  By default files are exported into the same \docs directory from which they were imported (CXE Docs Dir).

Character Encoding

Determines the character coding for export of the localized file.  For Microsoft Office and PDF files, the encoding will always be UTF-8.

UTF-8 Byte Order Mark (BOM)

Select method on how to process UTF-8 Byte Order Mark (BOM) information for original file.
  1.Use File Profile Settings
  2.Preserve; do not add if not originally present
  3.Preserve; add if not originally present
  4.Remove if present

Delay in Minutes

Set the delay between notification of Job/Activity cancellation and actual cancellation.

Buttons

 

Cancel

Cancels the export.

Export

Initiates the export source.  You will receive an email when the export is complete.
