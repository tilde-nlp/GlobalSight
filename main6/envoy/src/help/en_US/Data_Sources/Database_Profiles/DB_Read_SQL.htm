Database Read SQL

These screens are only viewable if you have a database integration.

 

Summary:

Write the SQL statements necessary to pull the data from the listed columns of a localizable row.

Read Connection:

Select the DB connection to use.

Read SQL:

Read SQL is a single SQL statement that reads all Localizable, Contextual and Invisible columns for a single record. The columns that are read must match the columns specified in the column information on the previous screen. Each localizable record is identified in the pending records table by its primary key. Use %%1%% to refer to the primary key. If the record uses a composite key, use %%1%% to refer to the first element, %%2%%, to refer to the second, %%3%%, to the third, etc. (see also Pending Records Table).

 

Example:

SELECT short_desc, long_desc, dept, prod_id, properties

FROM items

WHERE prop_id= %%1%% AND dept='%%2%%'
