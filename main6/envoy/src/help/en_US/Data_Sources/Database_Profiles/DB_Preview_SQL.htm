Database Preview SQL

These screens are only viewable if you have a database integration.

 

Summary:

Optional.  Write the SQL statements necessary to put back data for the listed columns of a localizable row for preview.

DB Preview Rules:

Enables rules for previewing database records in context.

Preview DB Connection:

Select the DB connection to use for writing data for preview .

Preview Insert SQL:

Preview Insert SQL inserts a new record into the specified database for preview if no record with that primary key already exists. If the record already exists, Preview Update SQL is used. This SQL statement should include all columns needed to necessary to define the record. Use %%1%% to refer to the first element of the primary key, and %%2%%, %%3%%, etc. to refer additional elements of a composite key (see also Pending Records Table). Use %%tablename.column%% to refer to the data needed for each column of the record. The tablename and column must be acquired as a part of the Read SQL and defined in the column information (see Columns).

 

Example:

INSERT INTO items(short_desc, long_desc, dept, prod_id, properties) VALUES ('%%items.short_desc%%','%%items.long_desc%%',

'%%2%%','%%1%%', '%%items.properties%%')

Preview Update SQL:

Preview Update SQL updates an existing record in the specified database for Preview if a record with that primary key already exists. If the record does not exist, Preview Insert SQL is used. This SQL statement is constructed in the same manner as the Preview Insert SQL above, employing the Update command instead of Insert, but must specify a Where clause to ensure that the correct record is updated.

 

Example:

UPDATE items

SET short_desc='items.short_desc',

long_desc='items.long_desc',

properties='%%items.properties%%'

WHERE dept='%%2%%' AND prod_id='%%1%%'
