Create/Edit File Profile

Summary:

Set the Encoding, File Format and Localization Profile to use for Importing content.

Name:

For display in GlobalSight. The name may only contain letters, numbers, the underscore "_" or the hyphen "-".

Description:

Optional, viewable by administrators and project managers in GlobalSight.

Localization Profiles:

Select the localization profile that should be used when importing this type of content.

Source File Format:

Select the format (see Supported Data Formats) for the source content.  This tells GlobalSight how to read the content.  Selecting Un-extracted File allows non-supported data formats to take advantage of the GlobalSight workflow.

XSL File:

For XML format only. Upload XSL file for previewing XML files in online editor and in context review tool. And for in context review tool, please use the XSL to FO file.

QA Filter:

Select the QA Filter which configured rules to do the QA checks.

UTF-8 Byte Order Mark (BOM):

Select method on how to process UTF-8 Byte Order Mark (BOM) information for original file.
 1.Preserve; do not add if not originally present
 2.Preserve; add if not originally present
 3.Remove if present

EOL Conversion:

Select how to convert the eol format on the exported file. For Plain text, Java properties and XML format only.
 1.Preserve original EOL format
 2.Convert to Unix-style format (LF)
 3.Convert to Windows-style format (CRLF)

Source File Encoding:

Select the encoding of the source content.  This is grayed out if the file format has a fixed encoding.

XML Rules:

Select the XML rules to use to determine what is localizable within the XML content  (see also XML Rules, Writing XML Rules).  Note that this selection is grayed out for file formats that are not imported as XML.

File Extensions:

When manually importing files, only files with the extensions selected are displayed for import.  This box allows multiple selections.

Default Export:

Primary Target Files: When automatic export occurs at the end of a workflow, primary target files are exported.  For manual export, the primary target files are the default selection.

Secondary Target Files: When automatic export occurs at the end of a workflow, secondary target files are exported.  For manual export, secondary target files are the default selection.

Terminology Approval:

No: Translated segments will not be populated into terminology after job is finished and exported, Default behavior.
Yes: Translated segments will not be populated into terminology after job is finished and exported.

Populate Source Into Un-Translated Target:

No: Default, for XLF/XLZ files, if segment is not translated, the original target content will be put back, keep unchanged as original in exported files.
Yes: For XLF/XLZ files, if segment is not translated, put source content into target in exported files.

Buttons

 

Close

Cancels the file profile creation/edit.

OK

Saves the new/modified file profile.
