Create/Edit Import Settings

These screens are only viewable if you have a database integration.

 

Summary:

Define a pending records table to monitor and set job size parameters (see also Pending Records Table).

Name:

For display in GlobalSight. The name may only contain letters, numbers, the underscore "_" or the hyphen "-".

Description:

Optional, viewable by administrators in GlobalSight.

Pending Records Table:

The name of the pending records table you wish to monitor.  If you have more than one pending records table, you'll need to create more than one DB import setting.

Records per Page:

List the number of records that should appear on a page.  In GlobalSight localization is done within the context of pages, which are usually individual files.  In the case of DB records, a page is simply a collection of records.  

Pages per Batch:

The number of pages that constitute a batch.  When the localization profile job size is set to by Batch, this determines how many pages (see above) are necessary before the job meets the size threshold and moves out of Pending.

Force Partial Page After:

A partial page is one that doesn't have the complete number of records per page (see above).  The page is considered complete regardless of the number of records after the listed number of days.

Database Connection:

Select the DB connection to use in reading the pending records table listed above.

Buttons

 

Close

Cancels import setting creation/edit.

OK

Saves the new/modified import setting.
