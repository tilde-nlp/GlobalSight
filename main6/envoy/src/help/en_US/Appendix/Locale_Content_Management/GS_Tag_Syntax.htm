GS Tag Syntax

GS tags are specified in XML syntax. Accordingly, all attributes have values, and all values are quoted. However, the HTML extractor will accept a simplified HTML-like syntax and internally normalize it to XML syntax. The simplified syntax does not require attribute values to be quoted and is otherwise explained below. Locale values have the form “en_US”.  Names will be uppercased within GlobalSight so they should be specified as uppercase and referred to as such.  Names also have the limitation of being 40-characters long, if a longer name is specified it will be truncated when added to GlobalSight.

 

<GS ADD="name" />

A position where a snippet can be inserted.

 

<GS DELETE="yes"> … </GS>

A region of deletable content.

 

<GS DELETE="anything"> … </GS>

Simplified syntax; any value of DELETE gets normalized to “yes” even if it is “no”.

 

<GS DELETE> … </GS>

Simplified syntax; the value of DELETE will be set to “yes”.

 

<GS DELETE="yes" DELETED="locale, locale, …"> … </GS>

A region of deletable content that has been deleted in certain locales. If the value of DELETED is empty, the region will be included in all locales.

 

The following tags should never be written in the page manually since they depend on the System 4 database and existence of (specific) snippets. Instead, use the Snippet Editor UI accessible from the Main Editor to insert snippets or delete content.

 

<GS NAME="name" ADDED="locale" />

Insertion of a generic snippet.

 

<GS NAME="name" ADDED="locale" ID="num" />

If ID > 0, insertion of a specific snippet. If ID = 0, the generic snippet is inserted.

 

To load snippets into the Snippet Library, a Snippet Import File must be created with snippets surrounded by the following tags.

<GS EXTRACT="name" [DESC="description"]> … </GS>

An extractable generic snippet.

 

<GS EXTRACT="name" locale="xx_XX" [DESC="description"]> … </GS>

An extractable specific snippet.

 

Nesting of GS Tags

GS Tags can be nested.

  • Snippets that are included by a GS ADDED Tag can themselves contain insert positions and deletable regions.

  • Deletable Regions can contain insert positions and deletable regions. If an outer region is deleted, embedded GS tags are ignored.

 

It is the responsibility of the source page and snippet author to ensure that positions refer to existing snippet names, that snippets and HTML content fragments consist of HTML block elements, and that insertion of snippets or deletion of regions produces valid HTML.
