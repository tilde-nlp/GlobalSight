Customizing Extraction Settings

You can customize extraction in GlobalSight by editing the properties files discussed below.

 

Customizing extraction lets you control the amount of data that is made available for translation. For example, many HTML files contain information about font sizes, background colors, image URLs, etc., that you do not want to translate. In fact, you do not even want the translator to see this data because it complicates the screen and makes the important, translatable data hard to find.

 

By editing the extraction properties files below, you can tell GlobalSight to ignore much of this non-relevant information. This makes the import process run faster, keeps your TM clean, and, most importantly, makes translation easier.

 

Suppressing the Extraction of Specific Segment Types

To suppress the extraction of specific segment types, edit the the Tags.properties file, located at <weblogic_home>/globalsight/properties/Tags.properties.

 

For example, if you do not want HREFs extracted,

  1. Open Tags.properties in a text editor.

  2. Delete the line:
    a.href:url-a,\

    Note:     You must delete the line, commenting the line out is not sufficient.
     

  3. Restart Weblogic

 

Suppressing the Extraction of Sections of Files

To suppress the extraction of sections of individual files edit the Diplomat.properties file, located at <weblogic_home>/globalsight/properties/Diplomat.properties. This allows you to markup source files so that only certain sections are extracted. Follow the instructions in Diplomat.properties.

 

Customizing the Extraction of Microsoft Office Documents

Microsoft Office Documents produce a large amount of segments that are not relevant to localization and therefore should not be extracted. For this reason, GlobalSight has special properties files to suppress the extraction of Microsoft Office Documents. The files WordExtractor.properties, ExcelExtractor.properties, and PowerpointExtractor.properties, located at <weblogic_home>/globalsight/properties, control the extraction of Word, Excel, and PowerPoint, respectively. By default these files define no tag attribute as extractable except for IMG.SRC. If you want more tag attributes to be extracted, you must add them to the respective file.

 

Styles.properties (found in <weblogic_home>/globalsight/serverclasses) also plays a role in extraction.

The following settings can be adjusted:

  • Style names that should be extracted or not. The style names are entered as keys with a value of "true" or "false". Note that colons in property keys must be escaped by a backslash ("\:").

  • The default extraction behavior for style names sharing a common prefix. Currently, all Microsoft Office styles starting with "mso" do not get extracted. This is to minimize the clutter we see today in Word, Powerpoint and Excel documents.

  • The default extraction behavior for unknown styles. Set to "false", which is a change from System 4 up to System 4.3.x where this setting was "true".

 

Note

These property files influence how much information gets extracted as localizable before segments are passed over to System 4. They are not related to the Localization Profile's "Excluded Item Types", which specifies which of the already extracted item types should not be offered for translation in the editors.

 

All changes are not active until after Weblogic is restarted.
