Working with Placeholder Tags

Placeholders represent native code that cannot be easily displayed in the downloaded file.  This code is protected to ensure that it is not corrupted during the localization process.  Placeholders can be thought of as formatting.  Whenever possible, placeholders should appear next to the same words in the localized text, as they did in the original source text.

 

When a segment is saved and closed using the Segment Editor, it is checked for placeholder errors.  If an error is found, placeholders are automatically enlarged and brightened so they are easier to see.  You should never need to create a placeholder manually.

 

If placeholders are missing, a collection of missing placeholders appears. You are instructed to copy and paste, or drag and drop, the supplied placeholders in their proper locations – using the source text as a reference for placement.  

 

If an extra unnecessary placeholder appears in the segment, it appears in red with strikethrough, and you are prompted to delete it.

 

If there is a problem with the order of the placeholders, the problematic tags are marked in Orange. You are prompted to move the placeholders so that they appear in the correct order.

 

You are not able to save a segment until all placeholder errors have been resolved.  However, since it is possible to edit the text directly (without opening the segment), you should run the document check at the before you upload the file to fix any errors and reduce problems while uploading.

 

In general, all formatting performed on a segmented offline file is lost on upload.  For specific source format segment types, such as HTML and DOC, certain formatting is displayed in Word.  This formatting can be added or removed from segments and includes bold, italic, underline, line-breaks, and non-breaking spaces.
