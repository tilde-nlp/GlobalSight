Placeholder Tags (Errors)

General principles

  • All tags that appear in the source must also appear in the target.

Exceptions: bold [b] and [/b], italic [I] and [/I], underline [u] and [/u], and non-breaking space [nbsp].

  • All tags must have an opening tag and a corresponding closing /tag.  

Example: This is an opening font tag: [f1], and this is a closing font tag: [/f1].

Exceptions: The [x] tag and the [nbsp] tag do not have closing tags.  The [x] tag represents a tag that cannot be deleted.  Usually this is because the first part appears in one segment and the last part appears in another segment.  Both are represented by [x].

Error Messages

“The following tags are missing:”

  • You need to check to see that all tags in the source are also in the target.

Source example: Here is a pretty[f1] cat [/f1].

Target example: Here is my translation of a pretty cat.

Resulting error message: The following tags are missing: [f1] [/f1].

How to fix this: Add the missing tags to continue: Here is my translation of a pretty[f1] cat [/f1].

  • You will also get this message when you need to check to see that each Placeholder tag has a left bracket [and a right bracket]

Source example: Here is a pretty [f1] cat [/f1].

Target example: Here is my translation of a pretty [f1 cat [/f1].

Resulting error message: The following tags are missing: [f1].

How to fix this: Add the incorrect bracket to continue: Here is my translation of a pretty [f1] cat [/f1].

“The following tags are unbalanced:”

  • You need to ensure that added tags are added in a pair.  You may add or delete these tags as needed, but you must add or delete them as a pair.  

Source example:  Here is a missing [b] cat [/b].

Target example: Here is my translation of a missing [b] cat.

Resulting error message: The following tags are unbalanced: [b].

How to fix this: Add the missing tag to continue: Here is my translation of a missing [b] cat [/b].

 

“The following tags cannot be added:”

  • You’ll get this message when any tag that is not in the source (except bold, italic, underline in HTML only) cannot be added into the target.

Source example:  Here is a missing cat.

Target example: Here is my translation of a missing [f1] cat[/f1].

Resulting error message: The following tags cannot be added: [f1]  [/f1].

How to fix this: Here is my translation of a missing cat.

  • You’ll also get this error message when you need to enter brackets into the text of a segment. Since the first bracket is reserved for GlobalSight Placeholder Tags, you must enter in a double-bracket (which is interpreted as a single bracket in the Main Editor).

Source example:  Here is a missing cat.

Target example: Here is my translation of a missing [cat].

Resulting error message: The following tags cannot be added: [cat].

How to fix this: Here is my translation of a missing [[cat].
