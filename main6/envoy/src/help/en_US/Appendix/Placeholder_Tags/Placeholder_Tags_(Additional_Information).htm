Placeholder Tags (Additional Information)

Definition

Placeholder tags are representations of html tags.  These tags are only used to represent html tags which may need to be moved as part of the localization process, such as links.

Example

I want to go to the store.

 

If all of the html tags were shown like this:

 

I want to go to the <a href="www.a-store.com">store</a>.

 

The html could be accidentally corrupted by forgetting a " or a >.  Placeholder tags are used to ensure code integrity:

 

I want to go to the [l1]store[/l1].

 

Notice that the html tag has both a beginning and an end.  In the above example, this denotes where the link should start and stop.  Error messages are shown to localization participants to ensure that they have put both starting and stopping tags, and that they have not deleted the tags altogether.

Optional Placeholder Tags

Bold: [b][/b]

Italic: [i][/i]

Underline: [u][/u]

Non-breaking space: [nbsp]

 

These tags can be added or removed as necessary to assist proper localization, as some languages cannot be made italic without compromising readability and others need non-breaking spaces as part of punctuation.

Tags Without Closing Elements

Notice that the [nbsp] tag does not have a closing element.  Likewise, if a localizable html tag covers more than one segment in the source material, it is represented as an [x] tag.  The [x] tag cannot be deleted, even if it represents a <b> tag which is simply covering more than one segment.
