Writing XML Rules

The XML rules file defines the rules by which an XML file with a given Schema is read by GlobalSight.

 

The rule file for the XML extractor is an XML document itself. A single rule file can include rules for multiple document types by using multiple rulesets. If the rule doesn’t define some of an element’s or an attribute’s extraction rule, they will be extracted according to the default rule. A rule file consists of four kinds of elements.

 

Rule file specification

<schemarules> element

This is the document root element. It contains one or more <ruleset> elements.

<ruleset> element

The <ruleset> defines the rules for a document type.

 

Attribute

Description

Required

schema

This is the name of the document root element. Note that the name is case sensitive.

Yes

 

The <ruleset> element contains one or more <translate> and <dont-translate> attributes.  You can use more than one <ruleset> element to define the rules for more than one document type (schema) in a single Rule file.

<translate> element

The <translate> element specifies elements or attributes that are to be extracted for the translation.

Attribute Description Required

 

Attribute

Description

Required

path

XPath expression to address the elements and attributes that are to be extracted for the translation.

Yes

loctype

Localization type. It specifies whether the extracted data are translatable or localizable. The possible values are “translatable” or “localizable”. The default value is “translatable”.  All of the words in a translatable item are counted toward the file’s wordcount.  Localizable items are given a wordcount of 1, regardless of the number of words.  Examples of localizable items include image source paths that may need to be changed, but should not be considered part of the wordcount.

Optional

datatype

Format of the data. If the extracted data needs further extraction, the data format should be specified in this attribute. The typical use case is that a HTML snippet is stored in an XML element. When the datatype attribute has a value “html”, the XML extractor extracts the content of the element and call the HTML extractor passing the extracted content.  Note that in order for HTML to be correctly interpreted within XML The ‘<’ and ‘>’ components of an HTML tag must be escaped in XML as &lt; and &gt; respectively.

Optional

type

Type of the data. This attribute is used when the type of the extracted data needs to be explicitly specified. These types can be excluded using the Exclude Item entry in Localization Profiles.  You can list an item type that already exists (see Exclude Item Types) or specify a new item type and exclude this item on an as needed basis.

Optional

inline

This attribute specifies if the elements specified by path attribute break a segment. If an element breaks a segment, the element tag is not included in the extracted data. If an element doesn’t break a segment (if the tag is in-line), the element tag is included in the extracted data. The possible values for the attribute are “yes” or “no”. “yes” means the tag doesn’t break segments. “no” means the tag breaks segments. The default value is “no”.  For example, if an element appears in the middle of a sentence it should be marked “yes” – otherwise the sentence would be broken into two pieces, split by the element, which would be linguistically incorrect.

Optional

containedInHtml

Use if the given element is contained within another element whose datatype is HTML. The possible values for the attribute are “yes” or “no”.

To use this tag, you will also need to edit Tags.properties in the properties folder, and set InlineTag_html and PairedTag_html to include the element name. This is so that the tag can be treated as if it were an HTML tag.

Additionally, you may want to edit LingAdapter.properties and set the unescapedTags property to include the name of the element. This is so that the tag can be unescaped when exported. For example if the tag is left as &lt;reg&gt; instead of <reg> when exported then this property should be set.

For example:
<paragraph>Although in an <bigBold>XML</bigBold> file, this contains pseudo-HTML.</paragraph>
would require adding the tag bigBold to the above mentioned property files.

Optional

movable

This attribute specifies whether these tags can be moved around in the editor.

Optional

erasable

This attribute specifies whether these tags can be deleted in the editor.

Optional

priority

If there are more than one rule that refer to the same element in the xml file, this attribute specifies a priority on the translate rule or dont-translate rule.

A smaller value will have a higher priority. The default value is 10 and negative can be used.

Optional

 

<dont-translate> element

The <dont-translate> element specifies elements or attributes that are not to be extracted. Use this element on any items extracted by default that are not to be translated.

XPath

The “path” attribute in above elements uses XPath expression. XPath is a language for addressing parts of an XML document. It is a W3C Recommendation. This document doesn’t discuss the specification of the language other than showing a couple of examples. The W3C Recommendation can be found at http://www.w3.org/TR/xpath. There is also a web page that lists examples of XPath expressions at http://www.zvon.org/comp/r/tut-XPath_1.html#intro.

Default rule

If an XML document or element is not associated with a rule, the default rule is applied to extract the document.

  • Contents of all the elements are extracted and counted as translatable.

  • All the element tags break segments. In other words, all the tags go into skeleton and no tag is included in an extracted translatable segment.

  • No attributes are extracted.

 

Character encoding

As with any XML document, a character encoding used in the rule file must be listed in the XML declaration unless the file is encoded in UTF-8.

Example:
XML Rule:

<?xml version="1.0"?>

<schemarules>

    <ruleset schema="sample">

        <!-- "aaa" element under sample is not extracted -->

        <dont-translate path="/sample/aaa"/>

        <!-- Any element that has "translate" attribute and the -->

        <!-- value of it is "no" is not extracted -->

        <dont-translate path='//*[@translate="no"]'/>

        <!-- Nothing under "bbb" is extracted -->

        <dont-translate path="/sample/bbb"/>

        <dont-translate path="/sample/bbb//*"/>

        <!-- All attributes in "ccc" is extracted -->

        <translate path="/sample/ccc/@*"/>

        <!-- "ccc" tag itself is extracted -->

        <translate path="/sample/ccc" inline="yes"/>

    </ruleset>

</schemarules>

 

XML File (use this file with the XML Rule above):

<?xml version="1.0"?>

<sample>

    <aaa>This is aaa text.</aaa>

    <bbb translate="no">This is bbb text.

        <bbb2>This is bbb2 text.</bbb2>

    </bbb>

    <ccc sampleAttribute="MySampleAttribute">

    This is ccc text.

    </ccc>

</sample>

 

The text in red will be translatable when you import the file into GlobalSight.

 

RELAX NG schema Used For XML Rule Files

<!-- RELAX NG schema for XML rule file -->

 

<grammar

  xmlns="http://relaxng.org/ns/structure/1.0"

  xmlns:c="http://globalsight.com/comment">

 

  <start>

    <ref name="schemarules"/>

  </start>

 

  <c:c> ------------ Element definition --------------------</c:c>

 

  <c:c> Document root </c:c>

  <define name="schemarules">

    <element name="schemarules">

      <optional>

        <ref name="use-empty-tag"/>

      </optional>

      <oneOrMore>

        <ref name="ruleset"/>

      </oneOrMore>

    </element>

  </define>

 

  <c:c>

       One ruleset element for each XML instance. There can be multiple

       ruleset elements in a rule file.

  </c:c>

  <define name="ruleset">

    <element name="ruleset">

      <attribute name="schema"/>

      <oneOrMore>

        <choice>

          <ref name="translate"/>

          <ref name="dont-translate"/>

          <ref name="dont-count"/>

          <ref name="sid"/>

          <ref name="internal"/>

          <ref name="comment"/>

        </choice>

      </oneOrMore>

    </element>

  </define>

 

  <c:c> Specify an element that is comment tag </c:c>

  <define name="comment">

    <element name="comment">

      <ref name="path"/>        <c:c> Mandatory attribute </c:c>

    </element>

  </define>

 

  <c:c> Specify an element that is internal text </c:c>

  <define name="internal">

    <element name="internal">

      <ref name="path"/>        <c:c> Mandatory attribute </c:c>

    </element>

  </define>

 

  <c:c> Specify an element that is to be translated </c:c>

  <define name="translate">

    <element name="translate">

      <ref name="path"/>        <c:c> Mandatory attribute </c:c>

      <optional>

        <ref name="loctype"/>

      </optional>

      <optional>

        <ref name="datatype"/>

      </optional>

      <optional>

        <ref name="type"/>

      </optional>

      <optional>

        <ref name="inline"/>

      </optional>

      <optional>

        <ref name="containedInHtml"/>

      </optional>

      <optional>

        <ref name="movable"/>

      </optional>

      <optional>

        <ref name="erasable"/>

      </optional>

      <optional>

        <ref name="priority"/>

      </optional>

    </element>

  </define>

 

  <c:c> Specify an element that is not to be translated </c:c>

  <define name="dont-translate">

    <element name="dont-translate">

      <ref name="path"/>        <c:c> Mandatory attribute </c:c>

      <optional>

        <ref name="inline"/>

      </optional>

      <optional>

        <ref name="priority"/>

      </optional>

    </element>

  </define>

 

  <c:c> Specify an element that is sid </c:c>

  <define name="sid">

    <element name="sid">

      <ref name="path"/>        <c:c> Mandatory attribute </c:c>

      <optional>

        <ref name="name"/>

      </optional>

    </element>

  </define>

 

  <c:c> Specify an element with words that are not to be extracted or counted </c:c>

  <define name="dont-count">

    <element name="dont-count">

      <ref name="path"/>        <c:c> Mandatory attribute </c:c>

      <ref name="words"/>        <c:c> Mandatory attribute </c:c>

    </element>

  </define>

 

  <c:c> ------------ Attribute definition --------------------</c:c>

 

  <define name="use-empty-tag">

    <attribute name="use-empty-tag">

      <choice>

        <value>true</value>

        <value>false</value>

      </choice>

    </attribute>

  </define>

 

  <c:c> XPath expression to select parts of XML documents </c:c>

  <define name="path">

    <attribute name="path"/>

  </define>

 

  <c:c> name: specify the sid name </c:c>

  <define name="name">

    <attribute name="name"/>

  </define>

 

  <c:c> words: specify the words that not to be extracted or counted </c:c>

  <define name="words">

    <attribute name="words"/>

  </define>

 

  <c:c> Localization Type: "translatable" or "localizable" </c:c>

  <define name="loctype">

    <attribute name="loctype">

      <choice>

        <value>translatable</value>

        <value>localizable</value>

      </choice>

    </attribute>

  </define>

 

  <c:c>

         Datatype: extraction format of the selected parts (html,

         javascript, xml, etc). Note that the values are in lower case.

  </c:c>

  <define name="datatype">

    <attribute name="datatype"/>

  </define>

 

  <c:c>

         Type: type of the segment. e.g. link, abbr, font-name,

         etc. Default is text.

  </c:c>

  <define name="type">

    <attribute name="type"/>

  </define>

 

  <c:c> Inline: segment breaking tag or not </c:c>

  <define name="inline">

    <attribute name="inline">

      <choice>

        <value>yes</value>

        <value>no</value>

      </choice>

    </attribute>

  </define>

 

  <c:c> ContainedInHtml: HTML tag surrounded by xml </c:c>

  <define name="containedInHtml">

    <attribute name="containedInHtml">

      <choice>

        <value>yes</value>

        <value>no</value>

      </choice>

    </attribute>

  </define>

 

  <c:c> Movable: movable tag or not </c:c>

  <define name="movable">

    <attribute name="movable">

      <choice>

        <value>yes</value>

        <value>no</value>

      </choice>

    </attribute>

  </define>

 

  <c:c> Erasable: erasable tag or not </c:c>

  <define name="erasable">

    <attribute name="erasable">

      <choice>

        <value>yes</value>

        <value>no</value>

      </choice>

    </attribute>

  </define>

 

  <c:c> priority: specify the priority on the rule </c:c>

  <define name="priority">

    <attribute name="priority"/>

  </define>

 

</grammar>
