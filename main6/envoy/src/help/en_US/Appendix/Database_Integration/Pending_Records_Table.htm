Pending Records Table

Summary

For each table containing content to localize a Pending Records Table needs to be created in

the Source database. Each Pending Records Tables is monitored by a DB Import Setting in

GlobalSight. When a new record is placed into the Pending Records Table, the DM Import

Setting calls a DB Profile to determine what content should be brought in to GlobalSight to

localize this record. The record is then deleted from the Pending Records Table.

 When inserting new records into this table, use the following values:

  • Task_Queue_ID – NULL. This field is auto-incremented as new records are inserted.

  • Record_Profile_ID – The ID of the DB Profile from GlobalSight. The DB Profile contains information about which fields in the record need to be localized, and what localization process to follow for the content.

  • Substitution_Parameters – The primary key of the record. If it is a compound key, use four pipes to separate the elements. For example, if your key consists of an Item_ID, and a Locale_ID, you would write the values for those keys 1234 |||| en_US. Although it does not matter what order the keys are written, if different methods are used to write the primary keys to this table, it is important that all methods write the key elements in the same order.

  • Loaded – 0. GlobalSight uses this field to determine if the content of the record has already be imported into GlobalSight. The initial value of 0 indicates that the record has not been imported. After import, this value is automatically changed to a 1, and if the value is 1 the record is erased. This keeps the GlobalSight footprint on your database very small.

 

Writing Records To the Pending Records Table (Triggers)

You can use any number of methods to write localizable records to the Pending Records Table.  One common method is to use triggers. The triggers are placed on tables containing localizable data. Whenever a source record is updated or inserted, the trigger inserts a new record into the appropriate Pending Records Table.

 

Another common method is to make use of the same mechanism that your company currently uses to place new items into tables. For example, if there is a customizable application for users to insert new products into the database, you could add a “Localize this Record” checkbox to this application. If the box is checked, when the application writes the new or changed record to the database, this record would also be placed in the appropriate Pending Records Table for localization. This method has the advantage of allowing users to specify that certain changes to the record do not require localization. For example, if the user updated a record to correct a spelling mistake, they could uncheck the box and the record would not be re-localized.
