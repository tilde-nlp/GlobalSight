What are UTF-8 and UTF-16

UTF-8 and UTF-16 are encoding methods for Unicode characters. UTF-8 is a multi-byte encoding, which uses 1 to 4 bytes per character. The 1-byte per character range, 0x00 to 0x7f, are identical to 7-bit ASCII. The bytes that represent all other characters (2 to 4 bytes per char) set the most significant bit on (8th bit is on). That makes parsing this encoding easier than the legacy multi-byte encodings. UTF-16 uses 16-bit as a character unit. The code point of this encoding is a direct mapping from Unicode code point. The exception is that the characters greater than 0xffff are represented as two 16-bit units, which are called surrogate pairs.

 

UTF-8 and UTF-16 are both Unicode encodings so they are suitable to use in multilingual environment.

 

UTF-8 is probably better in a situation where most strings are English or European languages because of its ASCII equivalent characteristics. UTF-16 has an advantage that all characters are represented as one 16-bit unit. But this advantage is lost when the surrogate pair is introduced.
