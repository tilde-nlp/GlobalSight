How does GlobalSight decide what is translatable in a page

The logic in each extractor decides what is translatable in a page. This logic has been developed over time, based on customer feedback. The logic is updated on an ongoing basis.

 

The logic in each extractor classifies all data in the given format into the following basic categories:

  1. Skeleton - Structural data for the document, never touched by the translator
  2. Code (intra-segment skeleton) - Skeleton data that is inside a segment
  3. Localizable
  4. Translatable - Data that needs to be translated
