Auto-Import Configuration

Summary

Auto-Import allows files from a file system to be automatically imported into GlobalSight.  GlobalSight must already be configured with any necessary file profiles before auto-import can be configured.

 

Set Auto-Import Parameters

1. Edit the file GlobalSight/applications/DefaultWebApp/properties/autoFileImport.properties

2. Use the table below to enter the correct parameters for your GlobalSight installation.

 

Note: It is not necessary to restart GlobalSight in order for the new entries in the auto-import files to be re-read.  This file is read dynamically.

 

Parameter

Description

disableAutoImport

Flag (true or false) to enable the entire Auto-Import feature. If it is false then no directories will be watched for auto-import.

 

Default: false

 

pollingFrequency

The polling frequency in seconds. This is how often the GlobalSight will poll the directories for changes.

 

Default: 60

 

BatchSize

This feature has been removed from the product.  The default value of "one" is correct.

 

Default: one

 

AssumeUnencounteredFilesAreNew

Flag (true or false) to tell GlobalSight whether to consider files in a directory that has not previously been scanned are new. For example, if the directory en_US\pressReleases exists and has five files before the first time GlobalSight is run, then all five of those files will be identified as new files and imported if the property is set to true. If the property is set to false, then the five files will not be imported unless they later get modified. A sixth file added to the directory will be identified as new and imported regardless of the setting.

 

Default: true

 

Specify Directories to Watch

1. Edit the file GlobalSight/applications/DefaultWebApp/properties/autoFileImport.map

2. Enter one line for each directory that you want to watch. Enter each line in the format:

<directory> | <jobNamePrefix> | <FileProfileName>

Where:

a. <directory> is the directory you want to watch.

b. <jobNamePrefix> This item is ignored.  The filename without the extension is always used as the job name.  You'll still need to enter a value; "xyz" will suffice.

c. <FileProfileName> is the File Profile in GlobalSight that corresponds to the files in this directory.

 

For example, the entry:

en_US\pressReleases | PressReleases | My en_US HTML Files

 

specifies that the directory <GlobalSight_docs>\en_US\pressReleases will be watched by GlobalSight for new and changed files. Jobs created from these imports will be prefaced with the words “PressReleases”. The files imported from this directory will be processed as specified in the GlobalSight File Profile “My en_US HTML Files”.

 

Note: It is not necessary to restart GlobalSight in order for the new entries in the auto-import files to be re-read.  This file is read dynamically.
