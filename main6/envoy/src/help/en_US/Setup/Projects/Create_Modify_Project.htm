Create/Edit Project

Summary:

Create a project with an assigned Project Manager, Termbase and Users. Project Managers can only edit projects they are assigned to, and may only create projects where they themselves are the project manager.  Administrators may edit any project.

Name:

For display in GlobalSight. The name may only contain letters, numbers, the underscore "_" or the hyphen "-".

Project Manager:

Select the user responsible for jobs assigned to this project.  The user must have project management permissions.  If this value is changed, all workflows currently assigned to this project will be assigned to the new project manager.

Termbase:

Optional, select a termbase.

Description:

Optional.

Auto-accept Translation Task:

If this option is checked, the Translation task will be auto-accepted if its participants are single.

Auto-send Translations Edit Report:

This option is not available unless "Auto-accept Translation Task" is checked. To check this option, the participant will receive the email include Translations Edit Report.

Auto-accept Review Task:

If this option is checked, the Review Only task will be auto-accepted if its participants are single.

Auto-send Reviewers Comments Report:

This option is not available unless "Auto-accept Review Task" is checked. To check this option, the participant will receive the email include Reviewers Comments Report.

Include Compact Tags:

This option is not available unless "Auto-send Reviewers Comments Report" is checked. To check this option, the Report columns("Source Segment" and "Target Segment") will include compact tags.

Auto-accept PM Task:

If this option is checked and the PM is the only activity participant, the task will be auto-accepted.

Check for not translated segments:

If this option is checked, will check if there are un-translated segments in current task when click "Task Completed" on activity detail page.

Save Translations Edit Report:

If this option is checked, Translations Edit Report files from offline upload will be saved to "<filestore>\<Company>\GlobalSight\Upload\<JobId>\<Locale>\Translations Edit Report" directory.

Save Reviewers Comments Report:

If this option is checked, Reviewers Comments Report files from offline upload will be saved to "<filestore>\<Company>\GlobalSight\Upload\<JobId>\<Locale>\Reviewers Comments Report" directory.

Save Offline Files:

If this option is checked, normal formats of files from offline upload will be saved to "<filestore>\<Company>\GlobalSight\Upload\<JobId>\<Locale>\Offline Files" directory.

Allow Manual Running of QA Checks:

If this option is checked, manual running of QA Checks and generating report will be allowed for QA-based activity in activity detail page. QA Checks Report will be saved to "<filestore>\<Company>\GlobalSight\Reports\QAChecksReport\<JobId>\<Locale>\<TaskName_TaskId>" directory.

Auto-accept QA-based Task:

If this option is checked, the QA-based task which has "QA Checks" checked from corresponding activity will be auto-accepted if there is only one participant selected.

Auto-send QA Report:

This option is not available unless "Auto-accept QA-based Task" is checked. The participant will receive the email with generated QA Checks Report when this option is checked.

Allow Manual Running of DITA QA Checks:

Unchecked default. If checked, non-DITA task acceptor of this project can manually run the DITA QA checks and generate report.

Auto-Accept DITA QA-based Task:

Unchecked default. If checked, when workflow comes to one "DITA QA Checks" activity, that task will be auto-accepted if there is only one available user.

Auto-Send DITA QA Report:

This option is not available unless "Auto-Accept DITA QA-based Task" is checked. If checked, the acceptor will receive the auto-accept mail with the DITA QA report as attachment.

Buttons

 

Cancel

Cancels project creation/edit.

Users...

Assign users to this project (See Project Users).

Save

Saves the new/modified project.
