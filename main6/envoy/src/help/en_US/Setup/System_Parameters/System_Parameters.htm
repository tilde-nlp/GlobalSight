System Parameters

Summary:

Change system-wide parameters.

Export Directory Name Style:

Locale - Files exported in folders according to target locale standard abbreviation, e.g. en_GB for English (UK)

Language - Files exported in folders according to language of locale, e.g. German (Germany) is exported to 'German'.  Not recommended if you have more than one locale with the same base language, e.g. German (Germany) and German (Austria).

Web Server Admin Email Address:

Some system errors generate email specifically for the administrator of the hardware on which GlobalSight is running.

Minutes to delay a re-import of in-progress data:

If a file that is part of an existing job is selected for import, the removal of the existing job and subsequent import and creation of the new job is delayed by the amount of time specified.  This feature must be enabled in a parameters file.

Replace Term Matches:

Sets the system to automatically replace terminology matches in the segment so the translator does not have to select the Terminology match manually.

Allow anonymous termbase access:

Allows the termbase to be accessed without logging into GlobalSight.  The termbase can then be accessed at http:\\servername:port#\tb.

Store Native Format Documents in Corpus:

Stores the documents in their native format in the corpus TM for reference.

Show all TMs in Corpus Search:

Allows users to select TMs to search in for Corpus Search.  Otherwise users are confined to the TMs selected in the TM Profile used to create the Job in which they are working.

Allow Localization Participants to use MT:

Allows localization participants to use machine translation (if enabled).

Allow Localization Participants to Upload Support Files:

Uploading support files is normally restricted to project managers and administrators.  This allows all participants, including workflow managers and localization participants to upload support files.

System Default Currency:

Sets the default currency used when displaying costs throughout the system.

 

Costing Options:

Enable Revenue and Expenses: Enables two separate values for each rate, the first can be used to track revenue (billed to client), the second to track expenses (cost from vendor).

Enable Expenses only: Only one value is specified per rate.

Comment Sort Order:

Descending Time Order: Comments are displayed from newest to oldest.

Ascending Time Order: Comments are displayed from oldest to newest.

Default Sorting: Comments are displayed by task, with the latest task first.  Within each task comments are displayed from newest to oldest.

Enable Email Authentication:

Enables the Email Authentication feature of GlobalSight. This feature should be enabled only when the mail server used by GlobalSight requires authentication.

Email authentication username:

The mail server user that will be used to authenticate all email sent from GlobalSight. This is a special mail server user created just for GlobalSight email authentication purposes. You must add this user to the mail server that you specified above. This parameter will be ignored if the above Enable Email Authentication parameter is disabled.

 

Note: The username is not the username of the sender of GlobalSight email since GlobalSight emails can be sent from multiple users.

Email authentication password:

The password for the above user. This parameter will be ignored if the above Enable Email Authentication parameter is disabled.

Records per Page - My Jobs:

Specifies the maximum number of jobs that appear on a page of the My Jobs display.

Records per Page - My Activities:

Specifies the maximum number of activities that appear on a page of the My Jobs display.

Optimize GlobalSight database daily at:

Specifies the time when the GlobalSight database is optimized to maximize performance.  Performance may be affected while the optimization is being run.

New Parameters

 

Enable Single sign-on:

Enables the SSO feature on login page and company page.

Buttons

 

Cancel

Cancels any changes to GlobalSight system parameters.

Save

Saves any changes to GlobalSight system parameters.  Note that a GlobalSight restart is required for changes to take effect.
