1. Overview

The purpose of DIPLOMAT format is to provide a well defined interface when GlobalSight Diplomat and GlobalSight Ambassador interact each other. DIPLOMAT, which is a XML compliant file format, defines a data structure that is being translated in Ambassador. Using DIPLOMAT format enables to extract the translatable and localizable text from the original data source without loosing the structure of the original data.

2. Specification

2.1. Encoding

DIPLOMAT files must be written in UTF-8. The encoding declaration in XML declaration will be ignored. The DIPLOMAT processor assumes that the files are always in UTF-8 and process them accordingly.

2.2. Element Definitions

The following table lists the different elements of a DIPLOMAT document:

<diplomat> The <diplomat> element is the document root. It contains zero, one or more <translatable> element, zero, one or more <localizable> element, zero, one or more <skeleton> element and zero, one or more <gs> element.

Mandatory attribute: version, locale and datatype.
Optional attributes: wordcount and targetEncoding.
<translatable> The <translatable> element is used to identify its content as a translatable text. It contains one or more <segment> elements.

Mandatory attributes: blockId.
Optional attributes: datatype, type and wordcount.
<segment> The <segment> element is used to identify the boundary of the segment in the <translatable> element. It contains zero, one or more <bpt> elements; the same number of corresponding <ept> elements; zero, one or more <it> elements; zero, one or more <ph> elements; zero, one or more <ut> elements; and zero, one or more <localizable> elements. If no <segment> element is present in <translatable> element, that means the <translatable> element is not segmented yet so the linguistic processor has the privilege to perform the segmentation on it.

Mandatory attributes: segmentId.
Optional attributes: wordcount.
<localizable> The <localizable> element is used to identify its content as a localizable string. It contains zero, one or more <bpt> elements, the same number of corresponding <ept> elements, zero, one or more <it> elements, zero, one or more <ph> elements and zero, one or more <ut> elements.

Mandatory attribute: blockId.
Optional attributes: datatype, type and wordcount.
<skeleton> The <skeleton> element is used to identify its content as skeleton (presentational or structural codes) of the document. It contains no other elements.

Mandatory attribute: none.
Optional attributes: none.
<gs> The <gs> element is used to identify the mark-up of the add/delete functionality. It contains zero, one or more <translatable> element, zero, one or more <localizable> element, zero, one or more <skeleton> element and zero, one or more <gs> element.

Mandatory attribute: none.
Optional attributes: extract, add, delete, added, deleted.

The following table lists the different elements of a TMX document used in <translatable> element:

<bpt> The <bpt> (Begin paired tag) element contains zero, one or more <sub> elements. It is used to delimit the beginning of a paired sequence of native codes. Each <bpt> has a corresponding <ept> element within the segment.

Mandatory attributes: i.
Optional attributes: type, x, erasable and movable.
<ept> The <ept> (End paired tag) element contains zero, one or more <sub> elements. It is used to delimit the end of a paired sequence of native codes. Each <ept> element has a corresponding<bpt> element within the segment.

Mandatory attributes: i.
Optional attribute: none.
<sub> The <sub> (Sub-flow) element contains zero, one or more <bpt> elements; the same number of <ept> elements; zero, one or more <it> elements; zero, one or more <ut> elements; and zero, one or more <ph> elements. It is used to delimit sub-flow text inside a sequence of native code, for example: the definition of a footnote or the text of a title in a HTML anchor element.

Mandatory attributes: none.
Optional attributes: locType, datatype and type.
<it> The <it> (Isolated tag) element contains zero, one or more <sub> elements. It is used to delimit a beginning/ending sequence of native codes that does not have its corresponding ending/beginning within the segment.

Mandatory attribute: pos.
Optional attributes: type, x, erasable and movable.
<ph> The <ph> (Place holder) element contains zero, one or more <sub> elements. It is used to delimit a sequence of native stand-alone codes in the segment.

Mandatory attributes: none.
Optional attributes: type, x, assoc, erasable and movable.
<ut> The <ut> (Unknown tag) element contains no other elements. It is used to delimit a sequence of native codes, about which the exporter has no information.

Mandatory attributes: none.
Optional attribute: x, erasable and movable.

2.3. Attribute Definitions

The following table lists the different attributes used in the elements of a DIPLOMAT document. The same attribute may be used with multiple elements, but will be either mandatory or optional depending on the specific occurrence.

version The version attribute is used in the <diplomat> element. It indicates the version of the DIPLOMAT format to which the document conforms.
locale The locale attribute is used in the <diplomat> element to specify the language for the document. Its value is a combination of ISO Language Code and ISO Country Code with a dash between them (e.g. en-US). More information about ISO Language Code and ISO Country Code can be obtained in the java.util.locale API reference in the JAVA SDK Documentation.
datatype The datatype attribute is used in the <diplomat>, <translatable>, <localizable> and <sub> elements. If it is used in the <diplomat> element, it specifies the type of data contained in the document. If it is used in the <translatable>, <localizable> or <sub> elements, it overrides the data type used in these elements. Its default value is "unknown".
targetEncoding The targetEncoding attribute is used in the <diplomat> element. It specifies the character encoding to be used when the document is exported to the original format. The name of encoding must be supported in JAVA.
assoc The assoc attribute (Association) is used to define whether an <ph> element is associated with the previous or the following text. Its value must be "p" (previous), "f" (following), or "b" (both).
i The i attribute (Internal matching) is used in the content markup to pair the <bpt> elements with <ept> elements. This mechanism provides TMX with support to markup a possibly overlapping range of codes, such as: "<B>Bold <I>Bold+Italic</B> Italics</I>".
pos The pos attribute (Position) specifies that an <it> element is actually the beginning or the end part of a paired code that has no correspondence in the segment. Its value must be an empty string, "begin" or "end".
type The type attribute specifies the kind of data a <bpt>, <ph>, <it> or <sub> element represents. i.e. bold, italic, underline, etc.
x The x attribute (External matching) is used in the content markup to match <bpt>, <ph>, <it> and <ut> elements between each <translatable> element. This mechanism facilitates the pairing of allied codes in source and target text, even if the order of code occurrence differs between the two.
blockId The blockId attribute specifies an identifier of the <translatable> and <localizable> elements. Each blockId must be unique in a file.
segmentId The segmentId attribute specifies an identifier of the <segment> elements in a <translatable> element. Each segmentId must be unique in a <translatable> element.
wordcount The wordcount attribute specifies the number of words in the <diplomat>, <segment>, <translatable> and <localizable> elements.
locType The locType attribute specifies the localization type in the <sub> element. It's value takes either "translatable" or "localizabel". If this attribute is omitted, it defaults to "translatable".
erasable The erasable attribute specifies that the element is erasable when the segment is translated. The possible value is "yes" or "no". The default is "no".
movable The movable attribute specifies that the element is movable when the segment is traslated. The possible value is "yes" or "no". The default is "yes".
extract The value of the extract attribute is the unique name of the snippet to create from the text between the tags.
add The value of the add attribute is the name of the type of snippet that can be added at that location.
delete If the delete attribute is present and set to 1, it will mean that the item can be deleted.
added The added attribute will be added to the tag of an item that was added in the editor and whose value will the name of the language for which it was added.
deleted The deleted attribute will be added to the tag of a deleted item after a delete takes place in the editor and whose value will be a comma separated list of languages from which the item has been deleted.

3. Usage

3.1. TMX elements

A piece of a translatable text (segment) often has native codes of the original document in it. <B> or <I> tags in HTML document are the examples. TMX (Translation Memory Exchange) format has been developed by OSCAR group of LISA to provide a standard method to describe translation memory data. DIPLOMAT uses TMX elements to delimit the native codes of various data formats in a universal way. Abstracting the data format with TMX allows a data processor to treat any data format in the same way.

For more information of TMX, see http://www.lisa.org/tmx/tmx.htm#content.

3.2 Example

<HTML>
<HEAD></HEAD>
<BODY>
<P>Hello, <A HREF="world.html">world.</A></P>
</BODY>
</HTML>

The above HTML will be transformed to a Diplomat format as follows.

<?xml version="1.0"?>

<diplomat version="1.0" locale="en-US" datatype="html">

  <skeleton>&lt;HTML&gt;
&lt;HEAD&gt;&lt;/HEAD&gt;
&lt;BODY&gt;
&lt;P&gt;</skeleton>
  <translatable blockId="1">
    <segment segmentId="1">Hello, <bpt i="1" type="HREF">&lt;A HREF=&quot;<sub locType="localizable">world.html</sub>&quot;&gt;</bpt>world.<ept i="1">&lt;/A&gt;</ept></segment>
  </translatable>
  <skeleton>&lt;/P&gt;
&lt;/BODY&gt;
&lt;/HTML&gt;</skeleton>
</diplomat>
